# 要件定義

## 1. プロジェクト概要

- バーチャルオフィス（Gather風）Webアプリケーション
- Next.js App Router + TypeScript + Supabase 構成
- リアルタイムコミュニケーション機能を想定

## 2. 認証システム要件

### 2.1 サポート認証方式
- **Google OAuth認証**: 永続的なアカウント
- **ゲストログイン**: 一時的なセッション

### 2.2 ゲストユーザー仕様
- **一時性**: ブラウザセッション限定、永続化なし
- **データ消失**: ログアウト・セッション終了で情報は完全に消失
- **再利用不可**: 同一ゲストアカウントでの再ログイン不可
- **制限事項**: 長期利用が必要な機能は利用不可

### 2.3 認証状態管理
- メモリベース状態管理（Zustand使用）
- ゲストユーザーのlocalStorage永続化は行わない
- ページリフレッシュでゲストセッションはリセット

## 3. 技術仕様

### 3.1 フロントエンド
- **フレームワーク**: Next.js App Router
- **言語**: TypeScript
- **UI**: shadcn/ui + Tailwind CSS
- **状態管理**: Zustand
- **データフェッチ**: TanStack Query

### 3.2 バックエンド
- **認証・DB**: Supabase
- **リアルタイム**: Supabase Realtime（予定）

### 3.3 アーキテクチャ
- DDD（軽量版）+ FDD + クリーンアーキテクチャ
- Feature-Driven ディレクトリ構造
